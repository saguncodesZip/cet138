What is Full‑Stack Development?

Full‑stack combines front‑end (what users see) and back‑end (servers, databases, APIs) to deliver complete web apps.

At a high level, a full‑stack developer understands how to build user interfaces with HTML, CSS, and JavaScript, and how those interfaces communicate with back‑end services via HTTP and JSON APIs. Even though this portfolio has no back‑end, it demonstrates how the layers fit together.

  • Front‑end: HTML structure, CSS presentation, JavaScript behavior.
  • Back‑end: Server logic (e.g., Node/Express), authentication, business rules.
  • Database: Persistence layer (SQL/NoSQL), schemas, queries.
  • APIs: The contract between front‑end and back‑end; typically REST or GraphQL.
  • DevOps & Deployment: Hosting, CI/CD, monitoring, environment variables.
Common Stacks
  • MERN: MongoDB, Express, React, Node
  • LAMP: Linux, Apache, MySQL, PHP
  • Django: Python, Django, PostgreSQL
  • .NET: C#, ASP.NET, SQL Server

Request Flow (Client ⇄ Server)

  1. User clicks a button in the browser (front‑end).
  2. JavaScript makes a fetch() call to an API endpoint.
  3. Server authenticates, runs business logic, reads/writes the database.
  4. Server returns JSON. Front‑end updates the DOM to show results.
